Agent Framework es el código — tú escribes la lógica del agente. Agent Service es la plataforma — Azure lo hospeda, escala y asegura. Puedes usar uno sin el otro, o combinarlos.

AIAgent + .AsAIAgent()

Clase central. Envuelve cualquier IChatClient y le agrega instrucciones, tools, middleware y orquestación.

IChatClient

🔀

Orquestaciones Built-in

Sequential, Concurrent, Handoff, Group Chat, Magentic — patrones listos para multi-agente.

Evolución de APIs

Chat Completions → Assistants → Responses

Tres generaciones de APIs de OpenAI. Agent Framework usa Chat Completions. Foundry Agent Service usa Responses.

2023 — Presente

Chat Completions API

La API más usada y estable. Es la que usa IChatClient y por tanto Agent Framework.

✅ Estable — Usa Agent Framework
2023 — 2024

Assistants API (v2)

Primera API con estado en servidor (Threads + Runs). Code Interpreter y File Search. Siendo reemplazada.

⚠️ Generación anterior
2025+

Responses API

Sucesor unificado. Web Search, MCP, Image Gen, Background Tasks. API interna de Foundry Agent Service.

🚀 El futuro — Usa Foundry
